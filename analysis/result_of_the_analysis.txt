Анализ работы приложения job5j_tracker.

Анализ был выполнен 01.04.2024 г.

Анализ был проведён, на мой взгляд, наиболее удобным инструментом jconsole и jmap.

Согласно данным Java Monitor JVM работает в режиме 14 Threads,
среднее CPU Usage не привышает 33,3 %,
кол-во классов 2,3К..2,4К Classes.

Объём используемой Heap при старте приложения составил 7..20 Mb.

Тест 1 (Создание и удаление 10000 шт. записей)
Объём Heap увелился до ~70 Mb, спустя некторое время обрушился до 7 Mb.
Приложение отработало без задержек.

Тест 2 (Создание и удаление 100000 шт. записей)
Объём Heap сохраняется ~70 Mb.
Приложение отработало без ощутимых задержек.

Тест 3 (Создание и удаление 1000000 шт. записей)
Объём Heap увелился до ~400 Mb.
!Завершения операции удаления ВСЕХ записей за 20 мин НЕ УДАЛОСЬ!
Стоит отметить, что при удалении мы наблюдаем скачёк использования Heap памяти вниз,
но по в течении времени объём опять нарастает при постоянном уровне CPU Usage
(В течении всех 20мин кол-во записей класса ru.job4j.tracker.Item в jmap сохраняется).

Тест 3.1 (Создание и удаление 10000000 шт. записей)
Добавление записей отработало.

Тест 4 (Создание и удаление 100000000 шт. записей)
При создании 100М записей удалось вызывать
"Exception in thread "main" java.lang.OutOfMemoryError: Java heap space: failed reallocation of scalar replaced objects".
